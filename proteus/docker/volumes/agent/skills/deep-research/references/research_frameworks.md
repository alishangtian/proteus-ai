# 研究框架指南

## 概述
研究框架是深度研究的核心方法论，提供结构化的问题分析和解决方法。

## 核心研究框架

### 1. 第一性原理分析框架
**适用场景**: 新技术评估、根本原因分析、创新解决方案设计

**步骤**:
1. **拆解问题**: 将复杂问题拆解为基本组成部分
2. **识别基本原理**: 找出每个部分的基本原理和约束条件
3. **重新构建**: 基于基本原理重新构建解决方案
4. **验证优化**: 测试并优化新解决方案

**示例**: 分析电动汽车电池成本问题
- 拆解: 电池 = 材料 + 制造 + 研发 + 供应链
- 基本原理: 材料成本(锂、钴)、能量密度、充放电循环次数
- 重构: 探索新材料(磷酸铁锂)、新制造工艺、回收利用
- 验证: 成本模型、性能测试、市场验证

### 2. 系统思维框架
**适用场景**: 复杂系统分析、多因素问题、长期趋势预测

**步骤**:
1. **定义系统边界**: 明确分析范围
2. **识别系统要素**: 找出所有相关要素
3. **分析要素关系**: 绘制因果回路图
4. **识别反馈循环**: 找出增强和平衡反馈
5. **模拟系统行为**: 预测系统动态变化

**工具**: 因果回路图、存量流量图、系统动力学模型

### 3. SWOT-PEST整合框架
**适用场景**: 战略规划、市场进入分析、风险评估

**框架结构**:
```
外部环境分析 (PEST)
├── 政治因素
├── 经济因素
├── 社会因素
└── 技术因素

内部能力分析 (SWOT)
├── 优势
├── 劣势
├── 机会
└── 威胁
```

**整合分析**: 将PEST因素映射到SWOT的机遇和威胁

### 4. 多源验证框架
**适用场景**: 信息可信度评估、事实核查、争议问题分析

**步骤**:
1. **来源收集**: 收集不同类型的信息来源
2. **可信度评分**: 按权威性、专业性、时效性评分
3. **交叉验证**: 比较不同来源的相同信息
4. **矛盾分析**: 分析矛盾信息的原因
5. **共识建立**: 基于证据建立可信结论

**来源分类**:
- Tier 1: 学术论文、官方统计数据、专利文档
- Tier 2: 行业报告、白皮书、专家访谈
- Tier 3: 媒体报道、分析文章、公司公告
- Tier 4: 社交媒体、论坛讨论、用户反馈

### 5. 决策树分析框架
**适用场景**: 多选项决策、风险评估、投资分析

**步骤**:
1. **定义决策节点**: 识别关键决策点
2. **识别可能结果**: 列出每个决策的可能结果
3. **评估概率和影响**: 估计概率和影响程度
4. **计算期望值**: 计算每个选项的期望价值
5. **敏感性分析**: 测试关键假设变化的影响

### 6. 场景规划框架
**适用场景**: 不确定性高的长期规划、战略风险管理

**步骤**:
1. **识别关键不确定性**: 找出影响最大的不确定因素
2. **定义场景轴心**: 选择2-3个关键不确定性作为轴心
3. **构建场景矩阵**: 创建2x2或3x3场景矩阵
4. **详细描述场景**: 为每个象限编写详细场景描述
5. **制定应对策略**: 为每个场景制定应对策略

## 框架选择指南

| 研究类型 | 推荐框架 | 关键产出 |
|----------|----------|----------|
| 技术评估 | 第一性原理分析 | 技术可行性评估、创新方案 |
| 市场分析 | SWOT-PEST整合 | 市场机会分析、进入策略 |
| 战略规划 | 场景规划 | 多场景战略、风险预案 |
| 问题诊断 | 系统思维 | 根本原因分析、系统解决方案 |
| 决策支持 | 决策树分析 | 最优决策建议、风险量化 |
| 事实核查 | 多源验证 | 可信结论、证据链 |

## 实施建议

1. **框架组合使用**: 复杂问题可能需要多个框架组合
2. **迭代应用**: 初步分析后可能需要调整框架或重新应用
3. **团队协作**: 不同框架适合不同专业背景的团队成员
4. **文档记录**: 记录框架应用过程和关键假设
5. **持续改进**: 基于结果反馈优化框架应用方法
