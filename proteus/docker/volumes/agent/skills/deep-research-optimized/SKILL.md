# 🔧 强制评估激活流程

## ⚡ 84%激活成功率的强制评估机制

### 三步强制评估流程
**步骤1 - 评估** (必须在响应中完成):
针对本技能，评估：
- 用户请求是否符合深度研究场景？
- 是否需要复杂分析或多源验证？
- 是否适合本技能的质量标准？

**步骤2 - 激活** (紧接着步骤1立即执行):
如果评估为"是" → 立即使用 `Skill(deep-research-optimized)`
如果评估为"否" → 说明"不需要深度研究技能"并提供替代方案

**步骤3 - 实现** (只有激活后才能开始):
只有在步骤2完成后，才能开始深度研究执行

### 关键原则
- **承诺机制**: 必须在响应中做出明确承诺
- **立即执行**: 评估后必须立即激活，不能跳过
- **不激活不执行**: 如果不激活技能，就不能执行深度研究

# 🎯 明确触发与失败条件

## ✅ 何时应该使用本技能

### 明确触发场景
1. **学术研究任务**: 需要严格学术标准、文献综述、理论框架
2. **技术评估任务**: 技术选型、架构评审、性能评估
3. **市场分析任务**: 市场规模、竞争格局、趋势预测
4. **战略规划任务**: 业务战略、产品规划、投资决策
5. **风险评估任务**: 项目风险、技术风险、市场风险

### 触发关键词模式
- "深度研究[主题]"
- "[主题]技术评估报告"
- "市场分析关于[领域]"
- "战略规划研究[目标]"
- "风险评估分析[项目]"

## ❌ 何时不应该使用本技能

### 明确失败条件
1. **简单查询**: 可以通过简单搜索直接回答的问题
2. **快速回答**: 不需要深度分析或多源验证的问题
3. **已有明确答案**: 事实性问题或有标准答案的问题
4. **不需要结构化报告**: 只需要简要回答或建议
5. **时间极度紧迫**: 需要立即回答，无法承受研究时间

### 拒绝执行示例
- "现在几点了？" → 简单事实查询
- "怎么重启电脑？" → 操作指南，不需要研究
- "1+1等于多少？" → 明确答案问题
- "简单介绍一下AI" → 不需要深度分析

# 📊 质量保证与评测系统

## 🏆 评测驱动质量保证

### 基准评测用例库
已建立5个典型场景的评测用例：
1. **技术趋势研究** - AI在金融风控中的应用
2. **市场进入分析** - 电动汽车充电桩市场
3. **学术文献综述** - 深度学习在医疗影像的最新进展
4. **战略规划研究** - 公司数字化转型路线图
5. **风险评估分析** - 新技术的采纳风险

### 质量阈值标准
- **优秀**: ≥8.5/10 - 直接交付，作为最佳实践
- **良好**: 7.0-8.4/10 - 轻微优化后交付
- **合格**: 6.0-6.9/10 - 重点改进后交付
- **需改进**: <6.0/10 - 重大修订或重新研究

### 评测维度
1. **研究设计** (15%): 问题清晰度、范围合理性
2. **信息收集** (25%): 源质量、覆盖广度
3. **分析质量** (30%): 逻辑严谨性、洞察深度
4. **报告质量** (20%): 结构清晰度、可视化
5. **实用价值** (10%): 建议可行性、影响潜力
